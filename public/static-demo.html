Step 1: Deploy the Backend Server

The backend server handles data processing, database operations, and API requests. You can deploy it using Docker or traditional methods.

Option 1: Using Docker (Recommended)
# Clone the repository
git clone <repository-url>
cd blackhole_core_mcp

# Build and start the containers
docker-compose up -d
Option 2: Traditional Deployment
# Clone the repository
git clone <repository-url>
cd blackhole_core_mcp

# Install dependencies
pip install -r requirements.txt

# Start the server
python app.py

Step 2: Update the Frontend Configuration

Once your backend is deployed, you need to update the frontend to connect to it.

In the index.html file, find and update the API_BASE_URL:

// API base URL - change this for production
const API_BASE_URL = 'https://your-backend-server.com'; // Replace with your actual backend URL

Step 3: Configure MongoDB

The application uses MongoDB to store data. You can use a local MongoDB instance or a cloud-based service like MongoDB Atlas.

Update the .env file with your MongoDB connection string:

# MongoDB connection strings
# Cloud MongoDB URI - for production use
MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/?retryWrites=true&w=majority

# Local MongoDB URI - for development and testing
MONGO_URI_LOCAL=mongodb://localhost:27017/
System Architecture

The Blackhole Core MCP system consists of the following components:

  • Frontend: HTML/CSS/JavaScript web interface
  • Backend API: Flask-based RESTful API
  • Database: MongoDB for data storage
  • Processing Modules: Image OCR, PDF processing, etc.
  • Agent System: AI agents for data analysis

All these components work together to provide a seamless experience for processing and analyzing multimodal data.
