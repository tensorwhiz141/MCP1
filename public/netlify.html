Netlify Deployment Guide

Frontend Deployed Successfully!

Your frontend has been successfully deployed to Netlify. To complete the setup, you need to deploy the backend API server.

Step 1: Deploy the Backend API Server

The frontend is just one part of the application. You need to deploy the backend API server to handle data processing and database operations.

You have several options for deploying the backend:

  • Docker Deployment: Use the provided docker-compose.yaml file to deploy the backend on any server that supports Docker.
  • Traditional Deployment: Deploy the backend on a VPS or cloud service using the provided deploy.sh script.
  • Serverless Deployment: Adapt the backend for serverless platforms like AWS Lambda or Google Cloud Functions.

Step 2: Update the API Base URL

Once your backend is deployed, you need to update the API base URL in the frontend code to point to your backend server.

In the index.html file, find the following line:

const API_BASE_URL = '';

Update it to point to your backend server:

const API_BASE_URL = 'https://your-backend-server.com';

Then redeploy the frontend to Netlify.

Step 3: Configure CORS on the Backend

To allow the frontend to communicate with the backend, you need to configure CORS on the backend server.

In the app.py file, make sure the CORS configuration allows requests from your Netlify domain:

CORS(app, origins=["https://your-netlify-domain.netlify.app"])

Step 4: Test the Connection

After deploying both the frontend and backend, test the connection by:

  1. Opening the frontend in your browser
  2. Checking the browser console for any CORS or connection errors
  3. Testing the health check endpoint
  4. Testing the various features (image processing, PDF processing, etc.)
Need Help?

If you encounter any issues with the deployment, check the following:

  • Make sure the backend server is running and accessible
  • Check that the API base URL is correctly set in the frontend
  • Verify that CORS is properly configured on the backend
  • Check the browser console for any errors
  • Review the backend server logs for any issues

For more detailed instructions, refer to the README.md file in the project repository.
